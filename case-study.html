HONK LIVE

Real-Time Mumbai Bus Tracker with Smart Leave-Time Intelligence
Dakshat Jain · HONKHACK 2026

1. Problem

Mumbai has 3.5 million daily BEST bus commuters. Every day, riders face the same question: "When do I leave?" Existing apps show static timetables, not live bus positions. Riders arrive 15 minutes early or miss buses entirely.

"I always reach the stop 15 minutes early because I never know if the bus is coming."

There is no free, real-time bus tracker for Mumbai today. The city's BEST bus network publishes schedules but not live positions.

2. Solution

HONK LIVE is a mobile-first web app that tracks buses in real-time and tells users exactly when to leave home.

FeatureHow It Works
Live Bus TrackingReal-time positions on Leaflet maps with route-colored polylines and numbered stop markers
Smart Leave TimeAuto-calculated from bus ETA minus walk time to stop, with 1-second live countdown
Home Location PickerLeaflet map + Nominatim geocoding to search any Mumbai address and auto-calculate walk time
Push NotificationsBrowser notification fires when leave time drops below 5 minutes (PWA-ready)
Trip CompletionAuto-detected via progress-wrap algorithm, distance check, and state detection
Halted Bus ModeShows parked buses with amber markers; auto-transitions when bus departs

HostingVercel (frontend) + Render (backend) + MongoDB Atlas

Key Algorithms

ETA: eta = (haversine_distance / bus_speed) × traffic_multiplier
Traffic multiplier: 1.0x off-peak to 1.45x during rush hour (8-10 AM, 5-8 PM).

Trip Completion: Three triggers — bus state === 'completed', distance < 0.2 km, or progress wrap (progress drops from >0.85 to <0.15).

Leave Time: leave_time = bus_arrival_time - (haversine(home, stop) / 5 km/h)

4. Design Decisions

DecisionRationale
Leaflet over Google MapsZero API cost — critical for free hosting at hackathon scale
Nominatim over Google GeocodingFree, no API key required, works globally
HTTP polling + WebSocket fallbackWebSocket breaks on slow mobile networks; polling ensures reliability
1-second countdown timerSmooth UX — users see the number change in real-time
Color-coded urgencyGreen (>6 min), amber (3-6 min), red (<3 min) — instant visual cue

6. Results & Outcomes

Test users confirmed the app eliminates "when do I leave?" anxiety. The 3-tap flow (browse route → pick stop → see countdown) provides instant clarity. Push notifications fire at the exact right moment. The countdown timer with color states provides visual urgency without reading text.

Total infrastructure cost: $0 — all services run on free tiers (OpenStreetMap, Nominatim, MongoDB Atlas, Vercel, Render).
